## 项目定位

- 项目目录：`C:\Users\t250c\Documents\全电脑台车-CAN协议\qt-upper-computer-ui`

## 关键业务规则

- 主界面炮孔图由 `.dp/.tl` 炮孔文件解析并渲染。
- 左臂、中臂、右臂的 H 点需要同时显示。
- 当前选中的臂 H 点为黄色，未选中的臂 H 点弱化为淡蓝色。
- H 点与目标孔重合时，局部放大图和主视图对应孔位都显示绿色命中状态。
- 局部放大图是主视图中 H 点附近的一比一局部放大，不是独立示意图。
- 鼠标当前用于模拟 H 点输入；真实链路接通后应由 CAN_TO_NET 过来的三个臂 H 点坐标驱动。
- 带尾巴的孔可以改入口点和尾巴末端；不带尾巴的孔只能改入口点。
- 施工痕迹记录中，孔打完后入口点应标棕色，尾巴痕迹应按实际推进深度和入口角度渲染。

## 钻孔深度逻辑

- P01 钻孔模式支持：基准底平齐、孔等长、按布孔图打孔。
- 当前重点验证孔等长和基准底平齐。
- 基准底平齐中，用户输入值是基准孔深，例如 4m 或 5m。
- 标定 Q 点时，应记录整个工作面的公共基准面。
- 每个孔开打一瞬间，应抓取当前孔入口基准，再与公共基准面比较，计算修正后的目标孔深。
- X 轴正方向是工作方向，大拇指方向为 X 正。
- 三个臂中应以最远离设备的 H 点 X 值作为公共基准面的候选。
- 斜孔不能只按 X 差直接加减，需要结合推进梁进孔方向做三角换算，目标是从俯视图或三维图看孔底平齐。

## CAN_TO_NET 和协议原则

- PC 上位机不直接支持 CAN，通过 CAN_TO_NET 透传模块中转。
- 上位机接收 CAN_TO_NET 数据用于监控和显示。
- 上位机发给 CAN_TO_NET 的是操作或设置参数，再转发给控制器。
- P2/P3 一类监控页原则上显示控制器侧回传值，让用户知道参数是否真正修改成功。
- 工人看的界面不显示“协议”“CAN”等工程术语。
- 协议已经覆盖的界面项用绿色标识，未覆盖但上位机已有 UI 规划的用蓝色标识。

## 重要源码位置

- `src/main.cpp`：启动参数、截图/自检入口、窗口启动方式。
- `src/MainWindow.cpp`：主窗口、顶栏、页面构建、CAN_TO_NET 接入、触摸键盘和大部分 UI 页面。
- `src/MainWindow.h`：主窗口状态和页面指针。
- `src/SitonDrillMainView.cpp`：炮孔图、H 点、三维/视图切换、炮孔编辑和局部放大图核心。
- `src/CanBusModel.cpp`：CAN 业务状态解析和显示数据模型。
- `src/CanToNetClient.cpp`：CAN_TO_NET TCP/Modbus 透传客户端。
- `tools/`：现场检查、模拟服务、UI 审计和演示脚本。

## 恢复和继续建议

- 继续开发先打开源码仓库，运行 CMake/Qt 构建。
- 接 CAN_TO_NET 现场数据前，先用 `tools/Start-CanToNetMockServer.ps1` 和 `--self-test-can` 验证链路。
- 接入真实数据后，优先核对三臂 H 点、推进梁当前位移、钻孔状态、操作台选择臂、当前孔深/剩余孔深。
- 鼠标模拟逻辑在真实 CAN 数据链路稳定后应逐步关闭或变为调试开关。
