算法PCB通信说明

本页只整理算法PCB自身的通信边界，不混入 NET_TO_CAN 或整车控制PCB 的实现职责。 按模块视角，算法PCB负责接收编码器/姿态原始量、标零操作指令和自平行初始化指令，完成算法计算，再向 NET_TO_CAN 和整车控制PCB 输出结果。 其中标零操作的业务语义是：对应 bit 从 0 变 1 -> 读取当前对应传感器值 -> 只保存该传感器零位到铁电一次；保持为 1 不重复保存，清 0 后再次置 1 才会再次标零。当前程序链路为：编码器/姿态原始量走 CAN1，NET_TO_CAN/HMI 指令走 CAN2，算法PCB输出通过 CAN2_SendX 发送。

对应程序工程：F:\工作\实际进展与验证\模型确定\MC_LCD - 7Control_V1.2\MC_LCD - 7Control_V1.2
