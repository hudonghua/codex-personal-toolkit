# 视觉标定节点：T_shell_cam 与 TagCenter_world 验证

**日期：** 2026-05-12
**对话引用：** [视觉标定与光心方案](fd3a37b7-18ba-4a80-8d45-da74457b67e0)
**主题：** 用外壳三点 Q1/Q2/Q3 和相机输出 tx, ty, tz 标定相机光学坐标系，最终验证视觉换算的 Tag 中心世界坐标是否接近全站仪实测值。

---

## 1. 当前核心目标

不要再直接比较：

    |Q1 - Center|  vs  camera distance

因为 Q1 是相机外壳上的点，不是相机光心 O。这个比较天然包含外壳点到光心的固定偏置，所以误差大不等于视觉 distance 一定错。

真正目标是建立完整链路：

    Q1/Q2/Q3 -> T_world_shell -> T_shell_cam -> TagCenter_cam -> TagCenter_world

最终验收：

    TagCenter_world ≈ Center_world
    error_mm = |TagCenter_world - Center_world|

其中：

    TagCenter_cam   = 相机视觉输出的 (tx, ty, tz)，相对相机光心 O
    TagCenter_world = 相机链路换算出来的 Tag 中心世界坐标
    Center_world    = 全站仪直接测出来的 Tag 中心世界坐标

目标误差希望控制在 30mm 内。

---

## 2. 三个坐标系

    World：全站仪世界坐标系
    Shell：相机外壳坐标系，由 Q1/Q2/Q3 建立
    Cam：相机光学坐标系，原点是光心 O，方向由视觉 tx/ty/tz 与全站仪数据拟合得到

外壳坐标系和相机光学坐标系不一定重合。原因包括：

    镜头光轴不一定垂直于外壳面
    镜头安装可能偏心或偏角
    传感器芯片可能和外壳基准不完全平行
    Q1/Q2/Q3 定义的是外壳姿态，不是光学姿态

所以不能要求：

    R_shell == R_cam

正确关系是求固定转换：

    T_shell_cam = inv(T_world_shell) * T_world_cam

它包含：

    1. 相机光心 O 在外壳坐标系里的位置
    2. 相机光学坐标系相对外壳坐标系的 yaw / pitch / roll
    3. 相机 X/Y/Z 轴相对外壳 X/Y/Z 轴的固定偏转

## 4. 重要发现

1. 开灯/关灯重复性极好：同一点两次测量 distance 差 < 1mm，说明系统稳定。
2. 只用左右6组单独拟合时，O 点完全跑偏（跑到 5 米外）：左右移动几何约束不足，必须前后+左右联合拟合。
3. roll ≈ 90 deg：相机坐标系的 X 轴与世界坐标系的 Z 轴（高度方向）近似对齐，符合安装姿态。
4. ty 几乎不变（12~15mm）：Tag 始终在相机水平线附近，y 方向偏移很小。
