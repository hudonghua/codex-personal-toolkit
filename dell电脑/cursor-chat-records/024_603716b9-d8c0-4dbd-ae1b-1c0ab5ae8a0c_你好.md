- **自带一套“能跑起来验证”的环境**：云端 VM 里可以 **构建、测试、甚至用桌面/浏览器做验证**（文档里提到 computer use、截图/视频/日志等产物）。
- **模型**：文档写明 Cloud Agents 用 **精选模型**，并且 **始终在 Max Mode**，**不能关掉 Max Mode**——所以你在本机选的 DeepSeek 之类，**不一定**会原样对应到云端那一次运行（以云端策略为准）。
- **费用与账号**：一般需要 **付费计划**，并会涉及 **按模型 API 计价、消费上限** 等（见文档 Billing 部分）。
- **密钥**：本机 `.env` 不会自动出现在云端；敏感配置通常要走 **Cloud Agents 的 Secrets** 等官方推荐方式。

### 本机（Local）大致是这些特点

- **直接对着你当前工作区**：改的就是你磁盘上的项目，适合 **快速迭代、调试本机环境、未 push 的本地状态**。
- **资源在你电脑上**：长任务、多任务会和你抢 CPU/内存。
- **模型选择**：一般更贴近你在 Cursor 里选的模型（例如你选的 DeepSeek）。

### 怎么选用一句话

- **要并行、要跑很久、要云端自己 build/test、要基于远程仓库开 PR**：倾向 **Cloud**。  
- **要立刻改手边工程、要本机调试、要完全本地上下文**：倾向 **本机**。
