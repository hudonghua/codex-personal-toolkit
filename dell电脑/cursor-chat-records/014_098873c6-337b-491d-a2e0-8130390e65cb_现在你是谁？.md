```text
|------|------|

| 反光 | 凸面高光会让本应是黑/白的区域**过曝** |
| 位姿含义模糊 | 位姿是贴在“哪个面”上？白面顶、黑面顶、还是中间参考面？标准库通常按**一个平面**算 |

- **俯视**为主，视角接近法向  
- **漫反射光**，少侧光、少镜面反光  
- 凹凸**很浅**（相对 Tag 尺寸和拍摄距离可忽略）
```

```text
这里的 **dz** 一般指：在**相机坐标系**里，Tag 原点到相机光心沿 **光轴方向（Z 轴）** 的分量，也就是 `solvePnP` / ArUco 位姿结果里 **平移向量 t 的第三个分量** `tvec[2]`。它和“画面里 Tag 看起来有多大”是一回事的两面——下面分原理说明。

## 1. 相机模型：像素 ↔ 射线

标定后，内参把像素变成方向（针孔模型）：

\[
s \begin{bmatrix} u \\ v \\ 1 \end{bmatrix}
=
\begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}
\begin{bmatrix} X_c \\ Y_c \\ Z_c \end{bmatrix}
\]

- \((u,v)\)：Tag 角点在图像上的位置  
- \((X_c,Y_c,Z_c)\)：该点在**相机坐标系**下的 3D 坐标  
- **\(Z_c\)**：沿光轴的“深度”；常说的 **dz** 就是 Tag 参考点对应的 **\(Z_c\)**（在 OpenCV 默认约定下常取 `tvec[2]`）

没有标定，只能得到方向，**得不到绝对米数**。

## 2. 为什么单目也能有“距离”？——靠 Tag 的已知边长

单目本身有尺度模糊，但 AprilTag/ArUco 给了**真实世界尺寸**：

- Tag 边长 = \(L\)（米）  
- 四个角在 **Tag 坐标系**里坐标已知，例如正方形在 \(Z=0\) 平面上：\((\pm L/2, \pm L/2, 0)\)

于是有 4 组 **2D–3D 对应点**，用 **`solvePnP`**（OpenCV `cv::solvePnP` / `estimatePoseSingleMarkers`）求旋转 **R** 和平移 **t**：

\[
\mathbf{P}_\text{cam} = R\,\mathbf{P}_\text{tag} + \mathbf{t}
\]

- **t**（`tvec`）：Tag 原点在相机系下的位置  
- **dz ≈ tvec[2]**：Tag 原点在光轴方向的深度  

**原理**：Tag 在图上占的像素越多 ↔ 在针孔模型里 **\(Z_c\)** 越小（越近）；占的像素越少 ↔ **\(Z_c\)** 越大（越远）。已知 \(L\) 就把“像素大小”绑到了“米”。

## 3. dz 和“直线距离”不是同一个量

| 量 | 含义 |
|----|------|
| **dz = tvec[2]** | 沿**相机光轴**的深度（相机系 Z） |
| **\(\|\mathbf{t}\|\)** | 相机原点到 Tag 原点的**欧氏距离** |
| **到 Tag 平面的距离** | 还要用法向：Tag 正对相机时，三者接近；Tag 倾斜时 **dz 会明显小于斜线距离** |

工业里常说“离相机多远”，要确认你用的是 **光轴深度 dz** 还是 **斜距 / 平面法向距离**。

## 4. OpenCV 里典型数据流（ArUco / AprilTag）

```
角点检测 → 图像点 (u,v)
         ↓
已知 Tag 边长 L → 物体系 3D 角点
         ↓
相机内参 K、畸变 dist → solvePnP → rvec, tvec
         ↓
dz = tvec[2]   （单位与 L 一致，L 用米则 dz 是米）
```

注意 **OpenCV 的 tvec 定义**：物体坐标变到**相机坐标**的平移；Tag 原点的 **\(Z_c\)** 就是 `tvec[2]`。

## 5. 误差从哪来（和“原理”也相关）

- **Tag 尺寸 L 标错**：深度按比例整体偏差（L 大 10%，距离也偏约 10%）  
- **内参/畸变不准**：尤其画面边缘、广角  
- **角点亚像素误差**：远距离时几个像素就能带来明显 dz 抖动  
- **Tag 不平、弯曲**：仍按平面 PnP，dz 会偏  
- **倾斜**：`tvec[2]` 是原点深度，不是“正对距离”  

## 6. 和双目 / 深度相机 的区别

| 方式 | 深度依据 |
|------|----------|
| **单目 + Tag（PnP）** | 已知平面几何 + 内参，解 **tvec[2]** |
| **双目 / ToF** | 三角测量或直接测距，不依赖 Tag 边长 |

Tag 的 dz 属于 **“有尺度的单目位姿估计”**，不是图像模糊里的“景深”。

---

**一句话**：OpenCV 的 dz 来自 **已知 Tag 边长 + 相机内参**，用 **PnP** 把 2D 角点还原成 Tag 在相机系下的 3D 位置，**tvec 的 Z 分量**就是沿光轴的深度；不是测“激光距离”，而是 **几何反投影 + 已知物理尺寸** 算出来的。
```

```text
所以：**`distance` = 相机到 Tag 原点的斜距（mm）**；**`tz` 只是沿相机光轴的分量**，不是斜距。

## 2. 全站仪侧：Tag 尺寸与姿态

用角点算边长，三个 Tag 都约 **280 mm** 量级（NE、Z 两边都 ~282–283 mm），说明 **Tag 物理边长应按 ~0.28 m 参与 PnP**；设成 0.10 m 会把距离整体按比例拉错。

角点 Z 从 ~1334 到 ~1051，差 ~283 mm → **Tag 相对水平面有倾斜**（不是严格竖直面），PnP 仍按平面算会有误差。

ID0 角点里 **右上 E=3398、右下 E=3117** 差很多，建议核对是否 **左右/右上右下抄错**；算尺寸时以几何闭合为准。

## 3. 相机 vs 全站仪：能直接比什么？

全站仪是 **NEZ 绝对坐标**；相机是 **相对相机的 t、R**。  
**没有相机在全站仪坐标里的位置 (N,E,Z)**，就不能直接把 `tz=4.16 m` 和某个 `center_Z` 比。

可以比的是：**Tag 之间的相对距离**（与坐标系无关，只要相机解算自洽）。

| 点对 | 全站仪中心距 (mm) | 相机 ‖Δt‖ (mm) | 偏差约 |
|------|-------------------|----------------|--------|
| 0 ↔ 1 | **1563** | \(\|(-1.53,-0.54,0.28)\|\) ≈ **1650** | ~6% |
| 0 ↔ 2 | **760** | **790** | ~4% |
| 1 ↔ 2 | **791** | **880** | ~11% |

说明：

- 相机 **三 Tag 相对几何** 和全站仪 **大体一致**（厘米～十几厘米级差别）。  
- 单 Tag 的 **`distance` 是否准**，还要相机站坐标 + 同一套 Tag 边长 L；目前只能说不离谱，不能单凭 center_Z 断言 4.2 m 对错。
```

```
## 5. 结论（简洁）

1. **`distance` 不是 dz**，是 **tx,ty,tz 的合成斜距（mm）**；你这里 **tz < distance** 正常。  
2. **斜距 ~4.2～4.5 m**，和之前说的 5 m 量级接近（差在斜距/平距、测的是 Tag 中心还是站心）。  
3. **三 Tag 间距** 与全站仪比，大约 **4%～11%**，说明解算 **整体可用**，但不是毫米级。  
4. 要验证 **单点距离是否准**：需要 **相机全站仪坐标 + Tag 边长 0.28 m + 标定**；只给 Tag 中心无法直接证 `tz`/`distance`。
```
