# FSJ_666 遥控器显示屏位图映射经验

适用工程：
- F:\工作\AI模型\Z20\一些心得.txt\FSJ_666(左右臂)

关键结论：
- 遥控器显示图标主逻辑在 Src\App_lcd.c 的 Disp_Rock_Pic()。
- 左右侧竖排按钮映射是正确的，不要轻易改动：
  - 左侧 4 个按钮来自 gDIBitV bit16~19，对应接收机 318.2.0~3。
  - 右侧 4 个按钮来自 gDIBitV bit20~23，对应接收机 318.2.4~7。
- 顶部 3 个按钮不是 gDIBitV，而是 NO1_BIT / NO2_BIT / NO3_BIT，对应 318.4.0 / 318.4.1 / 318.4.2。
- 顶部按钮后来补进了 Disp_Rock_Pic()，位置可通过顶部按钮块里的 vXX/vYY 和 vXX += 间距调整。
- 中间两排钮子需要按真实接收报文反推，不要只看旧代码注释。此次确认过的映射：
  - left top 1 -> 318.0.1，两档
  - left top 2 -> 318.0.0，两档
  - right top 1 -> 318.1.0 down / 318.1.1 up，三挡自复位
  - right top 2 -> 318.1.4 down / 318.1.5 up，三挡自复位
  - left top 3 -> 318.0.2，两档
  - left top 4 -> 物理上需要显示到第二排第 2 格，曾用 gDIBitV bit25 显示
  - right top 3 -> 最终按用户确认显示 gDIBitV>>8 bit6（即原始 bit14），且方向需要反相
  - right top 4 -> 318.1.3 down / 318.1.2 up，三挡自复位
- 当用户说“方向反了”，通常只需要把显示索引从 ? 1 : 2 改成 ? 2 : 1，不一定是位号错了。
- 当用户说“往中间靠一点”时，优先改 Disp_Rock_Pic() 里对应图标块的 vXX 起点和 vXX += 间距，不要动图标素材表。

操作经验：
- 这类文件多为 GBK 编码，避免直接用 apply_patch；优先复制到 ASCII 工作路径，用 GBK 编码读写后再覆盖回原路径。
- 改动后必须立刻回读关键片段，确认改动已同步到原文件，而不是只改了工作副本。
