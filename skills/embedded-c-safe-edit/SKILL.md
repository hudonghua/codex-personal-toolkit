# 嵌入式C代码安全修改工作流

这是一个完整的工作流程，用于安全地修改GBK编码的嵌入式C代码。

## 核心原则

1. **修改前必须备份**（带时间戳）
2. **GBK文件只能用Python修改**（禁用Edit/sed）
3. **大括号问题现场修复**（绝不退档）
4. **先理解后执行**（打印理解等确认）
5. **修改后立即验证**（不敷衍用户）

## 完整工作流程

### 阶段1：理解任务

当用户交代修改任务时：

```
📋 我的理解：

任务目标：
- [具体要实现什么功能]

需要修改的文件：
- [文件1]：[做什么修改]

实现方案：
- [步骤1]
- [步骤2]

关键执行周期/数据流程：
- [如果涉及定时任务，说明执行周期]
- [必须追踪到定时器中断，不能只看函数名]

❓ 这样理解对吗？
```

**等待用户确认后再执行！**

### 阶段2：备份文件

```bash
cp 原文件.c 原文件.c.bak_$(date +%Y%m%d_%H%M%S)
```

### 阶段3：用Python修改（保持GBK编码）

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

# 1. 读取GBK文件
with open('目标文件.c', 'rb') as f:
    content = f.read().decode('gbk', errors='ignore')

# 2. 执行修改
# （根据用户需求修改content）

# 3. 用GBK编码写回
with open('目标文件.c', 'wb') as f:
    f.write(content.encode('gbk'))
```

### 阶段4：验证修改结果

```python
# 读取修改后的文件
with open('目标文件.c', 'rb') as f:
    content = f.read().decode('gbk', errors='ignore')

# 检查关键内容
if '期望的内容' in content:
    print("✅ 修改成功")
else:
    print("❌ 修改失败，需要重新修改")
```

### 阶段5：报告结果

```
✅ 已备份：文件名.bak_20260503_130000
✅ 已修改：[具体改了什么]
✅ 已验证：[显示修改后的关键代码片段]
```

## 特殊情况处理

### 情况1：大括号配对问题

**绝对不能退档！必须现场分析修复！**

#### 分析步骤：

1. **读取文件并分析大括号配对**
```python
with open('file.c', 'rb') as f:
    content = f.read().decode('gbk', errors='ignore')

lines = content.split('\n')

# 统计大括号配对
brace_stack = []
for i, line in enumerate(lines):
    for char in line:
        if char == '{':
            brace_stack.append(i)
        elif char == '}':
            if brace_stack:
                open_line = brace_stack.pop()
                print(f"第{open_line}行的{{ 对应 第{i}行的}}")
```

2. **根据程序逻辑定位问题**
   - 看函数边界：函数结束的 } 后面应该是下一个函数
   - 看 if/else 结构：else 块结束后不应该还在 else 内
   - 看变量作用域：变量在哪里声明？在哪里使用？
   - 看代码逻辑：这段代码应该属于哪个块？

3. **用Python修复**
```python
# 添加缺失的 }
lines.insert(行号, '}')

# 或删除多余的 }
del lines[行号]

# 写回文件
content = '\n'.join(lines)
with open('file.c', 'wb') as f:
    f.write(content.encode('gbk'))
```

### 情况2：需要恢复备份

**必须先征得用户同意！**

#### 恢复流程：

1. **列出所有备份**
```bash
ls -lht 文件名.bak* | head -10
```

2. **询问用户**
```
⚠️ 发现问题：[具体描述]

可用备份：
1. 文件.bak_20260503_140000 - [包含XXX功能]
2. 文件.bak_20260503_120000 - [包含YYY功能]

❓ 需要恢复备份吗？如果需要，选择哪一个？
```

3. **等待用户确认后执行**
```bash
cp 用户选择的备份 原文件
```

## 禁止的操作

❌ **绝对不能做：**
- 使用 Edit 工具修改 GBK 文件
- 使用 sed 添加中文文本到 GBK 文件
- 不备份就修改文件
- 遇到大括号问题就退档
- 不经用户同意就恢复旧备份
- 修改后不验证就说"已完成"
- 凭函数名下结论（必须追踪到定时器中断）

## 追根溯源检查清单

涉及定时任务时，必须确认：
- [ ] 这个变量在哪个函数里累加？
- [ ] 这个函数在哪里被调用？
- [ ] 调用条件是什么？（如 if(gT0Flg==1)）
- [ ] 这个条件在哪里触发？（如定时器中断）
- [ ] 定时器配置是多少？（如1ms中断）

**不要凭函数名（如app10ms）下结论，必须追踪到实际调用处！**
