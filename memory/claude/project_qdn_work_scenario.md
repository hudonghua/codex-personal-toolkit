# 全电脑模型 - 实际工作场景

**项目代号**: QDN (全电脑)  
**讨论日期**: 2026-04-04  
**核心内容**: 自动定位钻孔的完整工作流程

---

## 一、项目核心目的

### 三者必须一致

```
炮孔图上的孔号位置（设计坐标）
    ↓
实际推进梁末端H点位置（真实坐标）
    ↓
显示屏上的显示位置（计算坐标）

要求：三者一致，误差 < 20mm
```

**目标**：
- 操作手根据显示屏判断是否到位
- 显示屏显示的位置 = 真实位置
- 操作手可以信任显示屏数据

## 二、完整作业流程

### 准备阶段

```
1. 进场（09:30）
   - 车辆就位，打支腿
   - 全站仪建站
   - 车体坐标系标定（Q1、Q2、Q3、Q4）
   - 验证Q4，误差 < 5mm

2. 加载炮孔图（10:00）
   - 操作员在显示屏上加载炮孔图
   - 炮孔图包含所有孔号的大地坐标
   - 例如：
     孔号1: [5000, 3000, 1500] mm
     孔号2: [5200, 3100, 1550] mm
     ...
     孔号50: [6500, 4000, 2000] mm

3. 初始标定（可选）
   - 用R点和T点标定offset_calib
   - 或者直接用上次保存的offset_calib
   - 如果有视觉，开始自学习
```

---

### 钻孔作业循环

```
循环开始（10:10）：

1. 系统显示下一个目标孔号
   - 显示屏：「目标：孔号1，坐标 [5000, 3000, 1500]」
   - 显示当前H点位置：[5015, 2990, 1520]
   - 显示误差：ΔX=+15mm, ΔY=-10mm, ΔZ=+20mm
   - 总偏差：27mm

2. 操作手控制臂架移动
   - 手动操作或半自动
   - 逆运动学计算关节目标值
   - 臂架移动到目标位置
   - 实时显示位置偏差

3. 实时显示位置偏差
   - 当前H点：[5005, 2997, 1502]（正运动学 + 修正）
   - 目标孔号1：[5000, 3000, 1500]
   - 偏差：ΔX=+5mm, ΔY=-3mm, ΔZ=+2mm
   - 总偏差：6mm
   - 显示屏：「✅ 已到位，可以钻孔」

4. 到位判断
   - 如果 |偏差| < 20mm：
     显示屏：「✅ 已到位，可以钻孔」
   - 如果 |偏差| > 20mm：
     显示屏：「⚠️ 未到位，继续调整」

5. 操作手确认并钻孔
   - 操作手目视确认位置正确
   - 按下钻孔按钮
   - 钻孔3-5分钟
   - 如果有视觉：学习更新offset_calib

6. 完成当前孔，移动到下一孔
   - 显示屏：「孔号1完成 ✅，下一个：孔号2」
   - 显示孔号2的坐标和当前偏差
   - 重复步骤2-6

循环结束：所有孔钻完（约4小时）
```

## 三、关键问题：如何保证三者一致？

### 问题分析

```
炮孔图坐标（设计）：[X1, Y1, Z1]
    ↓
控制器计算H点（理论）：[Xh_theory, Yh_theory, Zh_theory]
    ↓ 修正层
控制器修正H点（实际）：[Xh_corrected, Yh_corrected, Zh_corrected]
    ↓
显示屏显示：[Xh_display, Yh_display, Zh_display]
    ↓
真实H点位置：[Xh_real, Yh_real, Zh_real]

要求：
Xh_display ≈ Xh_real ≈ X1（误差 < 20mm）
```

---

### 没有标定的问题

```
炮孔图：孔号1在 [5000, 3000, 1500]
控制器计算：H点在 [5030, 2980, 1520]
真实位置：H点在 [5000, 3000, 1500]

显示屏显示：「偏差 ΔX=+30mm, ΔY=-20mm, ΔZ=+20mm」
操作手：「明明已经对准了，为什么显示还有偏差？」
操作手：不敢钻孔，怕打偏

问题：控制器计算不准！
```

---

### 标定后的效果

```
炮孔图：孔号1在 [5000, 3000, 1500]
控制器计算：H点在 [5030, 2980, 1520]（理论值）
修正后：H点在 [5000, 3000, 1500]（修正值）
真实位置：H点在 [5000, 3000, 1500]

显示屏显示：「✅ 已到位，偏差 < 5mm」
操作手：「完美对准，可以钻孔！」
操作手：信任显示屏，放心钻孔
```

## 四、标定方案在实际工作中的应用

### 第一层标定：车体坐标系

**作用**：
- 确保控制器知道车体在大地坐标系的位置和姿态
- 确保炮孔图坐标和控制器坐标在同一个坐标系

**如果不标定**：
- 炮孔图是大地坐标系
- 控制器是设备坐标系
- 两者对不上，无法定位
- 显示的位置和真实位置完全不一致

---

### 第二层标定：末端定位

**作用**：
- 修正控制器计算的H点位置
- 让显示的位置和真实位置一致

**如果不标定**：
- 显示屏显示：「偏差30mm」
- 实际：已经对准了
- 操作手：不敢钻孔，怕打偏
- 或者：操作手不信任显示屏，只靠目视

---

### 视觉自学习修正

**作用**：
- 持续优化H点位置精度
- 适应传感器漂移、温度变化

**效果**：
- 第1个孔：精度20mm
- 第10个孔：精度10mm（学习了9次）
- 第30个孔：精度5mm（学习了29次）
- 越钻越准！

**操作手感受**：
- 开始：「显示还行，基本能用」
- 中期：「显示越来越准了」
- 后期：「完全信任显示屏」

---

## 五、精度要求分析

### 项目目标：< 50mm

**实际需求分析**：
```
孔径：通常 40-50mm
孔间距：通常 300-500mm

精度要求：
- 理想：< 10mm（孔径的1/4）
- 良好：< 20mm（孔径的1/2）
- 可接受：< 50mm（孔径大小）

如果精度 > 50mm：
- 可能打到相邻孔的位置
- 影响爆破效果
- 操作手不敢用
```

---

### 标定方案可以达到的精度

```
初始标定（用R点和T点）：
- 精度：20mm
- 状态：可用

视觉学习10次后：
- 精度：10mm
- 状态：良好

视觉学习30次后：
- 精度：5mm
- 状态：优秀

视觉失效时（用最近学习值）：
- 精度：12-15mm
- 状态：可接受
```

**结论**：完全满足要求！

## 八、项目本质理解

### 不是简单的机械臂控制

**而是**：
- 炮孔图设计坐标
- 控制器计算坐标
- 真实物理坐标
- **三者的精确匹配**

### 核心价值

**解决的问题**：
- 让显示的位置和真实位置一致
- 让操作手相信显示屏的数据
- 让自动定位真正可用
- 提高钻孔效率和精度

**如果没有这个系统**：
- 操作手只能靠目视对准
- 效率低，精度差
- 容易打偏孔
- 影响爆破效果

**有了这个系统**：
- 操作手看显示屏就能判断
- 效率高，精度好
- 减少人工判断误差
- 提高爆破效果
