## 视觉在项目中的定位

- 视觉是**第五层**（运行时修正、独立校核、失准报警）
- **不替代**控制器正解主链
- 主结果仍来自：传感器 → 正解 → 零位基准层 → 传感器转换层 → 加工误差层 → 现场摆放层

## 靶标方案决策

**不推荐**：
- 点激光做主方案 - 角度敏感、容易出光斑、正面性太强
- 单个大平面 AprilTag - 对"小区域、远距离、现场工况"不是最稳主方案

**推荐**：
- 固定相机
- 推进梁上固定刚性靶标
- 靶标做成有空间层次的刚体特征
- 视觉先解靶标位姿，再换算到H点

**目标区域**：约280×280 mm，长距离8~12 m

## 相机选型

### 工业相机方向（优先）
- 海康 `MV-CH120-10GM`
- 镜头：10~12 mm
- 用途：单相机较大范围覆盖 + 目标识别

### 安防IPC相机（验证用）
- `DS-IPC-B12HV3-LA`
- 可做K/D标定和流程验证
- **不作为**最终高精度主视觉首选
- 原因：本质是安防IPC，分辨率和成像链稳定性有限

## 标定链（已确认顺序）

1. **相机内参标定**
   - 得到K（内参矩阵）
   - 得到D（畸变系数）
   - 必须固定：分辨率、焦距、对焦、曝光、白平衡

2. **相机与外壳固定关系标定**
   - 外壳上有3个点
   - 通过全站仪得到外壳在世界坐标系下的位姿
   - 求固定外参 T_shell_cam

3. **运行时统一**
   - 结合 T_world_shell + T_shell_cam + 视觉测得的目标相对相机位姿
   - 统一到世界坐标系

## K/D标定建议

- 拍20~30张棋盘格图片
- 角度要丰富：中心、四角、边缘都要覆盖
- 避免：模糊、过曝、强反光
- 验证R/T时：用未参与求解的独立点做回代验证

## 控制器接口

- 视觉输出给控制器：`valid` + `quality` + `H_visual`（或`DeltaH`）
- 不建议把大量中间视觉细节直接灌给控制器
