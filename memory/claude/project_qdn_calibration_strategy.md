**项目代号**: QDN (全电脑)  
**讨论日期**: 2026-04-04  
**核心内容**: 两层标定策略、视觉自学习修正、倾角补偿机制

---

## 一、两层标定策略

### 第一层：车体坐标系标定（基础层）

**目标**: 确认O点正确转换到大地坐标系

**硬件配置**:
- 车体上安装4个棱镜（Q1、Q2、Q3、Q4）
- 立体布置，刚性固定，不在臂架上
- 双轴倾角传感器

**标定方法**:
- 用Q1、Q2、Q3标定 → 得到R0和T0
- 用Q4验证 → 误差 < 5mm为成功
- 保存倾角基准值（roll_base, pitch_base）

**关键理解**:
- Q1、Q2、Q3、Q4都在车体上，与7个关节传感器无关
- Q4误差只能来自倾角传感器或棱镜位置测量误差
- 不能来自7个关节传感器

**运行时补偿**:
- 实时读取倾角传感器当前值
- 计算变化量：Δroll、Δpitch
- 用变化量补偿R：R_current = ΔR(Δroll, Δpitch) * R0
- T基本不变：T_current = T0
- 适应车体姿态变化（液压内泄、地面沉降）

**传感器不参与修正**:
- 不修改倾角传感器零点
- 只保存标定时刻的基准值
- 用倾角变化量动态补偿坐标变换

---

### 第二层：末端定位标定（应用层）

**目标**: 提高末端H点定位精度

**标定方法**:
- 选择隧道面2个已知点：R点和T点（现场不允许多点）
- 控制臂架移动到R点和T点
- 记录误差：error_R、error_T
- 计算初始固定偏置：offset_calib = -(error_R + error_T) / 2

**修正位置**: 第二层（坐标变换层）
```
H_ts = R_current * H_car + T_current（坐标变换）
H_corrected = H_ts + offset_calib（固定偏置修正）
```

**物理意义**:
- 补偿整个系统的系统性偏差
- 不修改传感器参数
- 不修改固定参数（18个）
- 只在坐标变换后加一个固定偏置

## 二、视觉自学习修正方案（核心创新）

### 核心思路

**传统固定偏置的问题**:
- 用R点和T点标定一次后固定不变
- 无法适应传感器漂移、温度变化
- 精度逐渐下降

**改进方案：动态自学习偏置**:
- 初始值：用R点和T点标定
- 运行中：视觉有效时，持续学习更新offset_calib
- 视觉失效时：使用最近一次学习的offset_calib
- 优点：越跑越准，平滑降级

**Why**: 视觉有时候不靠谱，但大部分工况可以看到末端点。把视觉的修正值放到固定偏置里，哪怕有几个点视觉拍不到或数据错误，至少还可以拿上次的值进行补偿。

---

### 实现方法

**方案A：增量学习**
```
逻辑：
- 计算视觉误差：delta = H_qt - H_ts
- 增量更新：offset_calib += learning_rate * (delta - offset_calib)
- learning_rate = 0.1（学习率）

优点：平滑更新，抗噪声
```

**方案B：滑动平均（推荐）**
```
逻辑：
- 保存最近N次视觉误差（N=50-100）
- 取平均：offset_calib = average(history)

优点：非常稳定，抗噪声能力强
```

---

### 异常检测

**目的**: 防止错误的视觉数据污染offset_calib

**检测规则**:
```
1. 绝对阈值：|delta| > 100mm → 丢弃
2. 相对阈值：|delta - offset_calib| > 50mm → 丢弃
3. 通过检测才用于更新
```

---

### 数据持久化

**策略**:
- 每更新100次，保存一次到EEPROM
- 或每10分钟保存一次
- 下次开机从EEPROM读取，继续学习

**优点**:
- 断电不丢失学习结果
- 越用越准

---

### 运行效果预测

```
10:00 - 进场标定
- offset_calib = [+15, -10, +20] mm
- 精度：约20mm

10:30 - 工作30分钟
- 视觉有效率90%，学习270次
- offset_calib = [+12, -8, +18] mm
- 精度：约10mm（视觉有效时）

12:00 - 粉尘大，视觉失效
- 使用最近学习的offset_calib
- 精度：约15mm（比初始20mm好）

13:00 - 视觉恢复
- 继续学习优化
```

## 三、修正层的完整架构

### 第二层：坐标变换 + 综合修正层

**所有修正都在这一层统一处理**:

```
输入：H_car（设备坐标系）

处理流程：
1. 倾角动态补偿
   - 读取倾角传感器
   - 计算R_current = ΔR(Δroll, Δpitch) * R0
   - T_current = T0（基本不变）

2. 坐标变换
   - H_ts = R_current * H_car + T_current

3. 固定偏置修正（自学习）
   - H_ts += offset_calib
   - 如果视觉有效且正常：更新offset_calib

4. 视觉修正（可选）
   - 如果需要实时视觉修正：
     H_ts += (H_qt - H_ts)

输出：H_corrected（大地坐标系）
```

---

### 三种修正方式的关系

**1. 倾角动态补偿（实时，必须）**
- 作用对象：R和T
- 更新频率：每个控制周期（10ms）
- 数据来源：倾角传感器
- 物理意义：适应车体姿态变化（液压内泄）

**2. 固定偏置修正（自学习，推荐）**
- 作用对象：H_ts
- 更新频率：视觉有效时持续学习
- 数据来源：视觉误差
- 物理意义：补偿系统性偏差，自适应优化

**3. 视觉修正（实时，可选）**
- 作用对象：H_ts
- 更新频率：视觉更新频率（1-5Hz）
- 数据来源：相机 + QT
- 物理意义：实时精确修正

---

### 开关控制

```
三个独立开关：
- USE_TILT_COMPENSATION：倾角补偿（建议常开）
- USE_FIXED_OFFSET：固定偏置修正（建议常开）
- USE_VISION_CORRECTION：视觉修正（可选）

灵活组合：
- 最简：只开倾角补偿
- 推荐：倾角补偿 + 自学习偏置
- 完整：倾角补偿 + 自学习偏置 + 视觉修正
```

## 四、倾角传感器修正的是R还是T？

### 答案：主要修正R（旋转矩阵），T基本不变

---

### 物理分析

**车体倾斜的本质**:
```
液压内泄 → 支腿下沉 → 车体倾斜

倾斜导致：
- 车体坐标系的方向改变了 → R改变
- 车体原点O的位置几乎不变 → T基本不变
```

**数值示例**:
```
场景：
- 支腿下沉5mm
- 车体倾斜0.11°
- H点距离O点5000mm

如果不修正R：
- H点位置误差 ≈ 5000mm * sin(0.11°) ≈ 10mm

如果不修正T：
- O点位置变化 ≈ 1-2mm
- H点位置误差 ≈ 1-2mm

结论：
- R的影响大（10mm）→ 必须修正
- T的影响小（1-2mm）→ 可以不修正
```

---

### 实现方式

**推荐方案：只修正R**
```
标定时：
- 保存R0、T0
- 保存倾角基准值：roll_base, pitch_base

运行时：
- 读取当前倾角：roll_current, pitch_current
- 计算变化量：Δroll = roll_current - roll_base
                Δpitch = pitch_current - pitch_base
- 更新R：R_current = ΔR(Δroll, Δpitch) * R0
- T不变：T_current = T0

坐标变换：
P_ts = R_current * P_car + T_current
```

## 五、完整的标定和运行流程

### 进场标定（10分钟）

**步骤1：车体坐标系标定**
1. 全站仪建站
2. 车辆就位，打支腿，等待稳定（10分钟）
3. 快速测量Q1、Q2、Q3、Q4（< 5分钟）
4. 用Q1、Q2、Q3标定 → R0、T0
5. 验证Q4 → 误差 < 5mm
6. 保存倾角基准值（roll_base, pitch_base）

**步骤2：末端偏置初始化（可选）**
1. 全站仪测量已知点R、T
2. 控制臂架移动到R、T
3. 计算初始offset_calib
4. 或者直接用上次保存的offset_calib

---

### 工作期间（4小时）

**每个控制周期（10ms）**:
```
1. 读取倾角传感器
2. 计算R_current（倾角补偿）
3. 正运动学计算H_car
4. 坐标变换：H_ts = R_current * H_car + T_current
5. 应用偏置：H_ts += offset_calib
6. 如果视觉有效：
   - 计算delta = H_qt - H_ts
   - 异常检测
   - 如果正常：更新offset_calib（滑动平均）
7. 输出H_corrected
```

**定期保存（每100次更新）**:
- 保存offset_calib到EEPROM

**实时监测**:
- 显示倾角变化量
- 显示offset_calib当前值
- 显示视觉学习状态

## 六、关键优势

### 相比传统方案

**传统固定偏置**:
- 标定一次，固定不变
- 无法适应变化
- 精度逐渐下降

**自学习偏置**:
- 持续学习，自适应
- 能适应传感器漂移、温度变化
- 精度越来越高

---

### 相比纯视觉修正

**纯视觉**:
- 视觉有效时很准
- 视觉失效时无法工作

**自学习偏置**:
- 视觉有效时：学习并更新
- 视觉失效时：用最近的偏置
- 平滑降级，不会突然失效

---

## 七、数据结构

### 修正层参数

```
标定基准：
- R0, T0：车体坐标系标定结果
- roll_base, pitch_base：倾角基准值

自学习偏置：
- offset_calib：当前偏置 [ΔX, ΔY, ΔZ]
- history[]：最近N次视觉误差（滑动平均用）
- update_count：学习次数

视觉数据：
- H_qt：视觉测量值
- vision_valid：视觉是否有效

运行时数据：
- R_current：当前旋转矩阵（含倾角补偿）
- T_current：当前平移向量
- H_ts：理论H点（大地坐标）
- H_corrected：修正后H点
```

---

## 八、关键理解

### 分层修正策略

```
第一层：正运动学（理论计算）
- 输入：7个传感器值
- 输出：H_car（设备坐标）

第二层：修正层（补偿各种误差）
- 倾角补偿：适应车体姿态变化
- 固定偏置：补偿系统性偏差（自学习）
- 视觉修正：实时精确修正（可选）
- 输出：H_corrected（大地坐标）

第三层：应用层
- 使用H_corrected进行控制决策
```

---

### 误差归因

**车体层（Q1-Q4）**:
- 只与倾角传感器有关
- 与7个关节传感器无关

**臂架层（末端H点）**:
- 与7个关节传感器有关
- 与18个固定参数有关
- 用自学习偏置补偿

---

### 不修正的内容

❌ 18个固定参数（机械结构）
❌ 7个关节传感器零点（认为准确）
❌ 倾角传感器零点（只保存基准值）

### 修正的内容

✅ R（旋转矩阵）- 用倾角变化量动态补偿
✅ offset_calib（固定偏置）- 视觉有效时自学习更新
✅ T（平移向量）- 基本不变（影响很小）

---

## 九、工作时长和场景

**工作时长**: 4小时/次

**倾角变化预估**: < 0.5°（4小时内）

**标定策略**:
- 标定一次，全程使用
- 实时监测倾角变化，自动补偿
- 视觉有效时持续学习
- 4小时工作时间不长，标定一次 + 实时监测足够

## 十、验证标准

**车体坐标系标定**:
- Q4误差 < 5mm：成功
- Q4误差 5-10mm：可用
- Q4误差 > 10mm：失败

**末端定位精度**:
- 单点误差 < 20mm：可接受
- 单点误差 < 10mm：良好
- 单点误差 < 5mm：优秀

**倾角变化监测**:
- < 0.2°：正常
- 0.2-0.5°：警告，已自动补偿
- 0.5-1.0°：建议重新标定
- > 1.0°：必须重新标定
