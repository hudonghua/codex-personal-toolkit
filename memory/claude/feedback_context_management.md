# 上下文管理约定

## 问题背景
当对话上下文过大时，会出现API 400错误：
```
请求格式异常，请精简上下文或减少 MCP 工具后重试
```

此时任何命令（包括 `/compact`）都无法执行，只能 `/clear` 清空对话。

## 约定规则
**我必须在对话较长时主动提醒用户压缩上下文**

**Why:** 
- 我无法主动执行 `/compact` 命令
- 一旦出现400错误，所有命令都无法执行
- 预防性压缩可以避免工作进度丢失

**How to apply:**
1. **触发条件（满足任一即提醒）：**
   - 连续对话超过30轮
   - 连续工作超过1小时
   - 大量代码读写操作后
   - 感觉上下文接近限制时

2. **提醒方式：**
   ```
   ⚠️ 建议压缩：对话较长，建议输入 /compact 压缩上下文，避免API错误
   ```

3. **用户操作：**
   - 用户手动输入 `/compact` 压缩
   - 或者让我保存进度到记忆后 `/clear`

## 预防措施
- 重要节点主动保存进度到记忆文件
- 完成阶段性工作后建议用户压缩或清空
- 不要等到报错才处理

## 恢复流程
如果已经出现400错误：
1. 用户只能 `/clear` 清空
2. 用户说"恢复记忆"
3. 我从记忆文件中恢复工作状态
