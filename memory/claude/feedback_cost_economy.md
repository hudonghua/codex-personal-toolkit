# 成本节制（token/费用敏感）

**规则：用户对花费敏感，要主动控制 token 消耗——尤其启动阶段的记忆读取仪式和代码探索。**

**Why:** 2026-05-31 用户反馈"还没开始你就花了我20元"——指启动时逐个全文读多份记忆文件 + 环境探索（误判平台、反复试探），任务还没开始就烧了不少钱。

**How to apply:**
- 启动记忆：先读索引 `MEMORY.md`，**只在与当前任务相关时**才全文读具体 feedback 文件；备份/编码/大括号这类纪律凭索引摘要先记住，真要改 GBK 源码时再回读细节。
- 探索：用一次精准的 grep / 批量读代替反复试探；能并行的工具调用合并到一条消息里发。
- 不重复读已读过的文件；不做与任务无关的大范围搜索；不为了"验证"再把刚写/刚编辑的文件读回来。
- 与 [[feedback_backup_discipline]] 等强制纪律有张力时：纪律照守，但用最省方式（读索引+按需回读）达成。
