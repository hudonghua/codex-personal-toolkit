4. **判断 TagSize 和 ID 映射有没有错**

如果 150mm、300mm 混用时某些 Tag 的距离比例明显不对，说明 `ID -> TagSize` 可能有问题。

5. **判断需要几个 Tag 才够稳定**

你可以比较：

```text
只用 3 个 Tag 求外参
用 4 个 Tag 求外参
用 4 个 Tag 求外参 + 留 1 个 Tag 做验证
```

如果 3 个不稳、4 个明显稳，说明以后现场也要保证至少 4 个高质量控制 Tag。

**但有个关键点**

最好不要只有 4 个 Tag。因为如果你用这 4 个 Tag 来求相机外参，又用这 4 个 Tag 来评估误差，结果会偏乐观。

更好的实验室布置是：

```text
4 个 Tag 用来求 T_world_cam
第 5 个或第 6 个 Tag 不参与求解，只做验证
```

这样你才能知道相机位姿算出来以后，能不能真的预测别的位置。

**实验室推荐方案**

```text
固定 5~6 个 Tag
全站仪一次性测每个 Tag 中心世界坐标
其中 4 个作为控制点
剩下 1~2 个作为验证点

相机移动 10~20 个位置
每个位置拍一张或多张
记录：
1. 相机位置编号
2. Tag ID
3. TagSize
4. center_x / center_y
5. tx / ty / tz
6. 重投影误差
7. 反求 T_world_cam
8. 验证 Tag 的世界坐标误差
```

**注意**

如果你只测 Tag 中心点，也可以做 `3D-3D 配准`，但至少需要 3 个不共线 Tag，4 个更稳。

如果你想用每个 Tag 单独求相机姿态，那就不只要 Tag 中心，还要知道 Tag 平面姿态，或者测 Tag 四个角点的世界坐标。

我的建议：实验室先走简单路线，**测 5~6 个 Tag 中心点，用 4 个求外参，1~2 个验证**。这能最快回答你的核心问题：不用 Q 点，靠现场固定 Tag 反求相机位姿，误差能不能压下来。

**具体做法**

假设全站仪已经测好了固定 Tag：

```text
Tag1_world = (X1, Y1, Z1)
Tag2_world = (X2, Y2, Z2)
Tag3_world = (X3, Y3, Z3)
Tag4_world = (X4, Y4, Z4)

```

每次相机移动后，OpenCV 会给你每个 Tag 在相机坐标系里的位置：

```text
Tag1_cam = (tx1, ty1, tz1)
Tag2_cam = (tx2, ty2, tz2)
Tag3_cam = (tx3, ty3, tz3)
Tag4_cam = (tx4, ty4, tz4)
Tag5_cam = (tx5, ty5, tz5)
```

然后用 `Tag1~Tag4` 求当前相机到世界的转换：

```text
Tag_cam -> Tag_world
求出 T_world_cam
```

再拿 `Tag5` 做验证：

```text
Tag5_world_calc = T_world_cam * Tag5_cam
```

最后和全站仪实测的 `Tag5_world` 比：

```text
dx = Tag5_world_calc.x - Tag5_world.x
dy = Tag5_world_calc.y - Tag5_world.y
dz = Tag5_world_calc.z - Tag5_world.z

error_mm = sqrt(dx*dx + dy*dy + dz*dz)
```

这个 `error_mm` 就是你要看的真实误差。

P03     | Tag5   | 200      | 900      | 80     | -35    | 120    | 148
```

第一种：参与求解的 Tag 误差。

```text

再看 Tag1~Tag4 自己的误差
```

这个叫拟合误差，只能说明这几个点能不能对齐，结果会偏好看。

第二种：不参与求解的 Tag 误差。

```text
Tag5/Tag6 不参与求解
只用来验证
```

这个才是最有价值的真实误差。
