0. 0x50 分包规则

0x50 是电脑共用下发 ID。B7=0x00 进入算法PCB标零；B7=0x51 进入整车PCB应急操作；B7=0xA1/0xA2 为低频标定/修正点下发。

1A. 车体姿态点下发

该界面固定 5 个点，界面约束 O/Q1/Q2/Q3/Q4 全部填完后才允许发送。电脑保存全站仪绝对 NEZ；下发算法PCB时统一转换为相对当前 O 的 dN/dE/dZ。N_O/E_O/Z_O 来源于本次界面输入的 O 点全站仪绝对坐标，由上位机保存，不由算法PCB生成。

通信方向CAN ID分包Byte定义说明
电脑 -> 算法PCB0x50B7=0xA1B0点序号0=O，1=Q1，2=Q2，3=Q3，4=Q4；固定 5 帧，收齐后算法PCB自动校验。
电脑 -> 算法PCB0x50B7=0xA1B1-B2dNsigned int16，低字节在前；dN=N_point-N_O，单位暂按 mm。N_O 是当前 O 点的绝对 N 值。
电脑 -> 算法PCB0x50B7=0xA1B3-B4dEsigned int16，低字节在前；dE=E_point-E_O。E_O 是当前 O 点的绝对 E 值。
电脑 -> 算法PCB0x50B7=0xA1B5-B6dZsigned int16，低字节在前；dZ=Z_point-Z_O。Z_O 是当前 O 点的绝对 Z 值。

1B. 机械误差修正点下发

该界面按臂维护采样池，支持新增、编辑、禁用/恢复采样点。上位机断电保存完整采样池；下发时只发送当前启用点，算法PCB只接收当前有效点集。H 点同样使用当前 O 点绝对坐标作基点，先由上位机计算相对量再下发。

通信方向CAN ID分包Byte定义说明
电脑 -> 算法PCB0x50B7=0xA2B0对象+点序号0x00-0x3E 左臂点 0-62；0x40-0x7E 中臂点 0-62；0x80-0xBE 右臂点 0-62。
电脑 -> 算法PCB0x50B7=0xA2B0完成标记0x3F 左臂发送完成；0x7F 中臂发送完成；0xBF 右臂发送完成。收到完成标记后算法PCB校验/拟合该臂。
电脑 -> 算法PCB0x50B7=0xA2B1-B2dNsigned int16，低字节在前；所有 H 点全站仪 NEZ 均先减当前 O：dN=N_H-N_O。
电脑 -> 算法PCB0x50B7=0xA2B3-B4dEsigned int16，低字节在前；dE=E_H-E_O。
电脑 -> 算法PCB0x50B7=0xA2B5-B6dZsigned int16，低字节在前；dZ=Z_H-Z_O。

1C. 算法PCB总反馈

不新增反馈 ID，复用 0x150 的索引帧。标定/修正界面只处理 B0=0xE0；施工显示界面继续处理原坐标索引。上位机收到反馈后，应在当前界面旁边显示“接收/校验/保存状态、错误原因、收到点数、最大误差”，用于提示本次下发是否成功。

通信方向CAN ID索引Byte定义说明
算法PCB -> 电脑0x150B0=0xE0B1对象0x00 车体姿态；0x01 左臂；0x02 中臂；0x03 右臂。界面据此把反馈显示到对应页面/对应臂。
算法PCB -> 电脑0x150B0=0xE0B2状态0x01 接收完成；0x02 校验通过；0x03 已保存/已应用；0x04 校验失败；0x05 数据不完整。上位机应显示为“已收到/通过/已保存/失败/未完整”。
算法PCB -> 电脑0x150B0=0xE0B3错误码0x00 无错误；0x01 点数不足；0x03 缺少必要点；0x04 两两距离超差；0x05 R/T残差超差；0x06 H点误差超差；0x07 拟合失败。上位机应翻译成中文错误提示。
算法PCB -> 电脑0x150B0=0xE0B4收到点数本次车体姿态点或该臂启用 H 点数量；界面显示“已接收 N 个点”。
算法PCB -> 电脑0x150B0=0xE0B5-B6最大误差signed/unsigned int16，单位 mm；按错误类型表示最大点间距离误差、R/T残差或 H 点误差。界面显示“最大误差 xx mm”。
算法PCB -> 电脑0x150B0=0xE0B7保留默认 0。
