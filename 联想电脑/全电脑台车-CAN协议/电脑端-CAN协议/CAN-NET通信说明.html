CAN-NET通信说明

本页定义 Qt 上位机和以太网转 CAN 模块之间的网络封装协议。CAN 业务含义仍以《电脑端通信说明》和《算法PCB通信说明》为准，本页只说明电脑怎样通过网口把 CAN 帧送上总线、怎样读回 CAN 帧。

模块 IP192.168.0.105以太网转 CAN 模块地址
端口500暂定服务端端口
传输TCP暂定；若厂家模块只支持 UDP，外层传输再改
职责透传CANCAN-NET 不解释 0x50/0x150 的业务含义

1. 通信链路

Qt上位机TCP Client
保存绝对 NEZ / 采样池 / 界面状态
→
以太网转CAN模块192.168.0.105:500
TCP Server，负责 CAN 帧转发
→
CAN总线算法PCB / 整车PCB / 显示屏
按 CAN ID 处理业务

上位机发出的网络数据必须能还原成标准 CAN 帧：CAN ID、DLC、8 字节数据。算法PCB返回的 CAN 帧也通过同一 TCP 连接回到上位机。

4. 发送示例

4.1 电脑下发车体姿态点

CAN 业务帧：0x50/B7=0xA1。上位机先用本次 O 点绝对坐标计算相对量：dN=N_point-N_O、dE=E_point-E_O、dZ=Z_point-Z_O。

字段值说明
CAN ID0x50电脑 -> 算法PCB统一下发 ID
CAN DATA[点序号, dN_L, dN_H, dE_L, dE_H, dZ_L, dZ_H, 0xA1]一帧一个点；点序号 0=O，1=Q1，2=Q2，3=Q3，4=Q4。
CAN-NETAA 55 01 01 seq 00 00 08 50 00 00 00 ... checksum把上面的 CAN ID 和 DATA 放入 CAN-NET 帧。

4.2 电脑下发机械误差修正 H 点

字段值说明
CAN ID0x50电脑 -> 算法PCB统一下发 ID
CAN DATA[对象点号, dN_L, dN_H, dE_L, dE_H, dZ_L, dZ_H, 0xA2]0x00-0x3E 左臂点，0x40-0x7E 中臂点，0x80-0xBE 右臂点。
完成标记B0=0x3F/0x7F/0xBF, B7=0xA2左/中/右臂发送完成，触发算法PCB校验/拟合。

5. 接收示例

算法PCB反馈不新增 CAN ID，复用 0x150/B0=0xE0。以太网转 CAN 模块收到该 CAN 帧后，通过 CAN-NET 帧类型 0x02 回传给上位机。

CAN字段值上位机显示
CAN ID0x150算法PCB -> 电脑
B00xE0识别为标定/修正总反馈，不按末端坐标处理。
B1对象0x00 车体姿态；0x01 左臂；0x02 中臂；0x03 右臂。
B2状态显示“已收到 / 校验通过 / 已保存 / 失败 / 数据不完整”。
B3错误码翻译成中文错误提示，例如点数不足、两两距离超差、R/T残差超差、H点误差超差。
B4收到点数显示“已接收 N 个点”。
B5-B6最大误差显示“最大误差 xx mm”。
B7保留默认 0。

6. 心跳与异常

项目建议规则说明
连接建立上位机启动后连接 192.168.0.105:500连接失败时界面显示“CAN-NET未连接”。
心跳空闲时每 1s 发送 帧类型=0x03模块可回同类型心跳；若厂家协议已有心跳，按厂家协议。
断线判断连续 3s 无心跳/无数据视为断线上位机停止标定下发，提示网络异常。
重连断线后每 1s 重试连接重连成功后不自动重发未完成标定批次，需用户重新下发。
发送失败TCP写入失败立即报错上位机保留本地数据，不修改“已下发/已同步”状态。
校验错误帧头/长度/校验错误直接丢弃可在日志里记录原始字节，便于排查模块或线路问题。

7. 约定与待确认

  • 当前端口暂定为 500，传输暂定为 TCP。
  • 如果厂家以太网转 CAN 模块已有固定帧格式，本页第 3 节封装格式需要替换为厂家格式。
  • CAN 业务层不受厂家网络格式影响，仍按 0x50 下发、0x150/B0=0xE0 反馈。
  • 全站仪绝对 N/E/Z 由 Qt 上位机保存；算法PCB只接收相对当前 O 的 dN/dE/dZ。
