# CAN1 三臂传感器 ID 分配表

**工程**：MC_LCD - 7Control_V1.2（200A 屏）  
**总线**：CAN1（35 芯 5/17 脚）  
**生成日期**：2026-05-23  

---

## 1. ID 编码规则

| 臂 | 基址（十六进制） | 计算公式 | 节点数 |
|----|------------------|----------|--------|
| 左臂 | `0x180` | `CAN_ID = 0x180 + nodeID` | 9 |
| 中臂 | `0x280` | `CAN_ID = 0x280 + nodeID` | 8 |
| 右臂 | `0x380` | `CAN_ID = 0x380 + nodeID` | 9 |

说明：

- `nodeID` 从 **1** 起编，不用 0（0x180/0x280/0x380 保留为基址段起点）。
- 左右臂各 **6 编码器 + 2 油缸位移 + 1 拉伸**；中臂少 1 个自由度 → **5 编码器 + 2 油缸位移 + 1 拉伸**。
- 第三组基址 `0x380` 为 **右臂**（口述「左臂 380」按惯例修正为右臂）。

**CAN1 传感器节点合计：26 个。**

## 6. 帧格式（待协议定稿，占位）

当前工程尚未定义各节点报文格式，建议统一约定如下（与现有 `CAN1_Get_Data` 8 字节缓冲兼容）：

### 编码器帧（node 1~6 或 1~5）

| 字节 | 含义 |
|------|------|
| D0~D1 | 角度 raw，uint16，低字节在前 |
| D2~D7 | 保留 / 状态 / 计数 |

换算：`Enc16_ToDeg(raw, zero_raw, ratio)` → `*_deg`（见编码规范）。

### 位移 / 拉伸帧（node 7~9 或 7~8）

| 字节 | 含义 |
|------|------|
| D0~D3 | 位移 raw，uint32 或 float，待硬件协议确认 |
| D4~D7 | 保留 / 状态 |

换算：`mm = (raw - zero_raw) * scale * sign`（走 `ARM200A_ZERO_CFG` 零位层）。
