# 04 关键源码定位

## 最应该先看的文件

`E:\AI_划时代\全电脑_算法PCB\正确资料_算法PCB_CAN协议\MC_LCD - 7Control_V1.2\Src\Arm200A_Kine.c`

用途：三臂正解、坐标变换、机构参数、拟合参数都在这里。

关键位置：

- `gArmLeftBase / gArmMidBase / gArmRightBase`：约第 193 行开始。这里定义三臂固定机构参数。

- `out->o = (0,0,0)`：中臂 `has_o0=0` 时的关键证据。
- `Arm200A_ArrayToFix()`：约第 1460 行。22 个几何拟合参数映射。
- `arm_fit_one()`：约第 1575 行。正解拟合逻辑。
- `Arm200A_AddSampleByCurrentInput()`：约第 2086 行。采样点加入拟合。

## CAN 逻辑文件

`E:\AI_划时代\全电脑_算法PCB\正确资料_算法PCB_CAN协议\MC_LCD - 7Control_V1.2\Src\App_usr.c`

用途：CAN 收发、编码器原始值到角度、方向比例、协议 ID。

关键事实：

- 当前方向比例 `gArm*Ratio` 都是 `+1.0f`。
- 原始编码器大致按 `0..65535 -> 0..360 deg`。
- 当前在线计算里 `gArmUseZeroComp=0`，零点结构体里的 sign 暂未参与在线修正。
- 如果实际编码器方向与右手法则相反，当前直接可用的修正方式是把对应 ratio 改为 `-1.0f`，但改前应现场验证。

## 当前机构勾勒

左/右臂：

`O0 -> O1 -> O -> B -> A -> C -> D -> D1 -> E -> E1 -> F -> G -> G1 -> H`

中臂：

`O -> B -> A -> C -> D -> D1 -> E -> E1 -> F -> G -> G1 -> H`

其中当前中臂 O 是 Body 原点。如果新可见 O 成为 Body 原点，中臂也不应再默认 `O=(0,0,0)`，除非新可见 O 就是旧中臂 O。
