三模块 CAN 关系总览

依据同目录三份协议（算法PCB通信说明 / 整车PCB通信说明 / 电脑端通信说明）汇总，呈现 电脑端 · 整车PCB · 算法PCB 三者在两条物理 CAN 总线上的收发关系、共享 ID 仲裁与数据闭环。

三节点 · 两总线 · 关系拓扑

电脑端发 0x50/0x71/0x75/0x7A；收 0x150/0x152/0x153/0x154/0x15A/0x170/0x176
整车PCB发 0x170/0x176/0x17E；收 0x50(0x51)/0x71/0x75/0x7A/0x153
算法PCB发 0x150/0x152/0x153/0x154/0x15A；收 0x50(0x00)/0x17E/编码器链
编码器/倾角发 0x181~0x189·0x192~0x199·0x1A1~0x1A9·0x28F → 算法PCB
读图要点：整车PCB 挂在 CAN1（执行器总线），算法PCB 对电脑的显示数据走 CAN2（算法PCB对外链）；而 0x153 与 0x17E 让整车PCB↔算法PCB 在 CAN1 上直接互联。电脑与两块板各自构成一来一回的闭环。

共享 ID 与分包仲裁

0x50 —— 一个 ID，两个目标，靠 B7 分流

B7 值处理方忽略方物理总线用途
0x00算法PCB整车PCB 忽略CAN2(NET_TO_CAN链)三臂编码器/深度/倾角 标零
0x51整车PCB算法PCB 忽略CAN1(执行器)屏幕应急操作（原老协议 0x51 已并入 0x50）
仲裁规则：同一 0x50 由电脑发出，两块板都会收到，但各自只认自己的 B7 分包号，对方分包直接丢弃——以 B7 实现“一帧双投、各取所需”。

0x153 —— 一帧两用，深度给电脑、方向给整车

字节消费方内容
B0-B5电脑（显示）左/中/右臂实时凿岩深度 uint16（= max(原始-深度零位,0)）
B6-B7整车PCB（执行）三臂自平行方向位：摆上/摆下/摆左/摆右

两条核心数据闭环

闭环 A · 测量显示链（编码器 → 算法 → 电脑）

编码器原始角度与车体姿态进入算法PCB → 解算出末端坐标/夹角/深度/掉线状态 → 经 0x150/0x152/0x153/0x154/0x15A 上送电脑显示。操作员在电脑端按 0x50(B7=0x00) 标零，算法PCB 把当前编码器/深度/倾角存为基准——构成“测量—显示—校准”闭环。

闭环 B · 自平行控制链（整车 ⇄ 算法）

整车PCB 发 0x17E 让算法PCB 锁定当前姿态为自平行初值；算法PCB 持续用实时角度算出应摆方向，经 0x153 的 B6-B7 回送整车PCB 驱动臂架——构成“初始化—计算—执行”的实时自平行闭环。两条链路均在 CAN1 上完成。
